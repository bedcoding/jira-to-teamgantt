# Jira to TeamGantt

Jira 이슈와 TeamGantt 작업을 한 곳에서 모아 비교하고, 기간별로 텍스트로 뽑아내는 Chrome 확장 프로그램입니다.

## 기능

- **Jira 수집**: 현재 열려 있는 Jira 이슈 목록 페이지에서 키, 제목, 상태, 담당자, 업데이트 시간을 긁어 누적 저장. GraphQL 응답을 가로채는 API 모드와, GraphQL이 안 잡힐 때 쓰는 DOM 폴백 두 가지
- **TeamGantt 수집**: TeamGantt 프로젝트의 작업을 API 응답(`v1/projects/{id}/children`)을 가로채 정규화해서 저장. 본인 작업만 필터링 가능. List 뷰에 보이는 행만 긁는 DOM 폴백도 제공
- **비교**: 두 출처의 데이터를 Jira 키 기준으로 매칭. 매칭/Jira만/TeamGantt 고아(키있음/없음) 4가지 종류로 분류해서 한눈에 정합성 확인
- **단축키 자동 입력 (`Cmd+Shift+X` / `Ctrl+Shift+X`)**: 비교 탭에서 매칭 안 된 Jira 항목 중 다음 1건을 TeamGantt 활성 입력창에 자동 주입. Enter 확정은 안전을 위해 사용자가 직접. 다음 단축키를 누르면 직전 항목은 "저장됨" 으로 승격되어 큐가 자동 진행
- **추출**: 선택한 기간에 들어가는 항목을 `[KEY] 제목` 형식 텍스트로 모아 한 번에 복사
- **백업/복원**: 모든 데이터·설정을 JSON 파일로 저장하고, 미리보기 후 복원

## 설치

1. 이 저장소를 clone 하거나 ZIP 다운로드 후 압축 해제
2. Chrome 주소창에 `chrome://extensions` 입력
3. 우측 상단 **개발자 모드** 켜기
4. **압축해제된 확장 프로그램을 로드합니다** 클릭
5. `jira-to-teamgantt` 폴더 선택

상단 툴바에 아이콘이 보이면 클릭해서 팝업을 엽니다.

## 탭별 사용법

### 지라

1. **경로**: 본인 Atlassian 도메인 (예: `mycompany.atlassian.net`)
2. **기간**: 단위 토글(일/주/월/분기/년) 선택 후 `‹ 이전 / 이번 / 다음 ›`으로 이동. JQL의 `updated >= ... AND updated < ...` 부분이 함께 갱신됨
3. **JQL**: 기본 템플릿은 `assignee = currentUser() AND updated >= "..." AND updated < "..." order by updated DESC`. 필요하면 직접 수정
4. **최종** URL을 클릭하면 새 탭에서 해당 검색 페이지가 열림
5. 그 페이지가 떠 있는 상태에서 **Jira 수집** 클릭 → 현재 화면에 보이는 이슈가 chrome.storage에 누적됨

### 간트

1. **프로젝트** / **사용자**: ⚙️ 버튼으로 모달을 열어 ID·이름 목록을 등록. 보통 한 번만 세팅
   - 사용자 ID 얻는 법: TeamGantt의 People 페이지에서 개발자도구 Network 패널을 열고 `v1/companies/{id}/resources/company` 응답을 통째로 모달에 붙여넣으면 자동 파싱
2. **TeamGantt 수집 (API)**: 해당 프로젝트 페이지를 새 탭에서 열고, 확장이 API 응답을 가로채 저장
3. **TeamGantt 수집 (DOM)**: API 가로채기가 안 통할 때 폴백. 현재 List 뷰에 보이는 task 행을 직접 긁어 저장. 화면에 보이는 행만 수집되므로 좌측 사용자 필터를 미리 걸어두는 것을 권장
4. 본인 ID가 설정돼 있으면 본인이 담당자로 지정된 작업만 추려서 저장

### 비교

1. 위쪽 **검색** 으로 키·제목 부분일치 필터링
2. **날짜** 셀렉트로 행마다 표시할 날짜 기준 선택 (Jira 업데이트 / Jira 마감일 / TeamGantt 시작일 / TeamGantt 종료일)
3. ⚙️ 버튼은 **키 추출 규칙**(정규식) 편집. TeamGantt 작업 제목에서 Jira 키를 뽑아내는 규칙이라 팀의 제목 컨벤션이 일반적이면 손댈 일 없음
4. 행 색상: 흰색 = 양쪽 매칭, 빨강 = TeamGantt에만 있고 키 없음 등으로 시각 구분
5. 키 셀 클릭 → Jira 이슈 새 탭, 제목 셀 클릭 → 클립보드 복사
6. 하단 **데이터 관리** 섹션에서 탭별 백업/전체 삭제

### 추출

1. **소스**: Jira / TeamGantt 중 선택
2. **기간**: 지라 탭과 동일한 단위 토글 + 이전/이번/다음 패턴
3. 자동으로 결과가 텍스트 영역에 쌓임. **복사** 버튼으로 클립보드에 옮겨 보고서·메신저에 그대로 붙여넣기

### 단축키

비교 탭에서 매칭 안 된 Jira 작업을 TeamGantt에 한 건씩 자동 주입할 때 사용합니다.

- **mac**: `Cmd+Shift+X`
- **그 외**: `Ctrl+Shift+X`

다른 확장과 단축키가 충돌하면 `chrome://extensions/shortcuts` 에서 변경할 수 있습니다.

사용 흐름:

1. TeamGantt 프로젝트 페이지의 새 작업 입력창에 커서를 둠
2. 단축키 → 비교 탭의 다음 미매칭 Jira 항목이 `[KEY] 제목` 형식으로 주입됨
3. Enter로 확정 (이름/날짜 보정은 평소처럼)
4. 다시 단축키 → 직전 항목은 큐에서 빠지고 그다음 항목이 주입됨

## 데이터 위치

모든 데이터는 `chrome.storage.local` 에만 저장됩니다. 외부 서버로 전송하는 부분 없음. 백업/복원도 로컬 JSON 파일 단위. 동작 도메인은 `*.atlassian.net`, `app.teamgantt.com`, `api.teamgantt.com` 세 곳뿐.
